applovinMax.*

Type Library
Revision Release 2021.3646
Keywords ads, advertising, AppLovin, AppLovin Max
Platforms Android, iOS

Overview

The AppLovin Max plugin allows developers to monetize users and use mediation through AppLovin Max static interstitial, video interstitial, and rewarded video ads.

Notes

  • For the legacy non Max plugin click [here][plugin.appLovin]

  • The main api differences between legacy are: Sdk Keys are moved into project settings, platform specific unitIds are required in applovinMax.load(), and a few have phases changed.

  • Once you are registered, obtain your SDK key from the AppLovin developer portal. From the Account section, expand the Account menu on the left side, select Keys, and your SDK key should be revealed.

  • Native ads are not supported at this time.

  • Apps are automatically added to the AppLovin developer portal when you include this plugin and app keys within your project, require() it, and call applovin.init().

  • If you use the plugin and make changes to the ad settings in the AppLovin developer portal

Important

Check out new setHasUserConsent method to enable GDPR data collection restrictions.

Registration

To begin, please register with AppLovin Max. Once you have access to the AppLovin developer portal, you can view your apps, select ad preferences, and more.

AppLovin
local applovinMax = require( "plugin.applovinMax" )

Project Settings

To use this plugin, add an entry into the plugins table of build.settings. When added, the build server will integrate the plugin during the build phase. Also add your Sdk keys for Android and/or iOS as seen below.

    android =
    {
        applicationChildElements =
